## Context Anchors

> Things that are hard to reconstruct cold.

- `<dialog>` nested inside outer `<form>` — the inner `type="submit"` naturally fires the outer form. Only works when there's a single POST action; multi-action pages need a separate inner `<form>` (see `wo/[id]/confirm` for that pattern).
- `use:enhance` close-on-result shape: `use:enhance={() => ({ update }) => { confirmDialog?.close(); return update(); }}` — fires after the server response, closes dialog on both success (redirect) and error.
- `wo_fully_shipped` SQL in `shipping.js:318` — the `EXISTS (SELECT 1 FROM work_order_lines ...)` guard prevents WOs with zero lines from being falsely flagged as fully shipped.
- `$derived.by(() => { return ...; })` is the function-body form of runes derived; `$derived(expr)` is the expression form. Plain `$derived(() => ...)` assigns the function itself — silent bug.
