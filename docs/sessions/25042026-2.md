## Context Anchors

- `$derived(data.X)` not `const { X } = data` — plain destructure from a prop is a one-time snapshot; after a named action redirects back to the same route, data prop updates but stale consts don't. Use `$derived` for any value that needs to stay live.
- `use:enhance` result handler: omitting `update()` opts out of SvelteKit's default redirect-following. Always `await update()` unless you explicitly want to suppress navigation.
- `formaction` on submit buttons overrides the parent form's `action` — the correct solution for "one form, two server actions" (nested `<form>` is invalid HTML and is silently stripped by browsers).
- `requireAdmin` from `$lib/auth.js` returns a `fail()` response directly — `return denied` not `return fail(500, ...)`.
- `revertShipment` in `src/lib/services/shipping.js` — bare `UPDATE shipments SET status = 'DRAFT'`, no inventory reversal needed.
- `back` action in `src/routes/inventory/count/+page.server.js` — non-mutating action, just returns counts map for form repopulation.
