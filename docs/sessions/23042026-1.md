## Decisions Made

- **`getCountBalancesAsOf` uses `run_date` for consumption** (not `created_at`) — run_date is when material is physically consumed; confirmations typically happen the next business day
- **No early confirmation** — a production run cannot be confirmed before its `run_date`; enforced in `confirmRun()` service
- **Balance as of = same definition as matrix** (receipts + adjustments − consumption); "pre-production" framing abandoned after discussion
- **Inventory count detail page** shows both "Counted Sq Ft" (`counted_sqft`) and delta (`sqft_quantity` with sign) side by side
- **`LEAST(run_date, created_at)` rejected** — proposed to handle early confirmations, but user chose to block early confirmation at the source instead
- **`getBalancesAsOf` PO query** should use `expected_date` only (not `OR status = RECEIVED`) for point-in-time accuracy
