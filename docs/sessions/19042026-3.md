## Decisions Made

- Customers table: name + phone only — no addresses on the customer record itself
- Customer addresses stored in `customer_addresses` lookup table (per-customer address book),
  linked to WO at ship time by copy (denormalized on WO) so historical address is preserved
- Contacts: separate from customers, linked to work_orders (not customers); multiple per WO;
  independent of customer relationship
- Shipment numbering: `{so_number}-S{N}` (e.g. `WO12345-S1`)
- One invoice per shipment (handled by external app — this app produces billing summary only)
- Billing summary: sqft by SKU only, no dollar amounts
- PDF output: deferred — current solution is print CSS + browser print-to-PDF
