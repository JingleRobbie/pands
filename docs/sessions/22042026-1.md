## Decisions Made

- **Partial roll split model**: When fewer rolls are shipped than a run contains, the original run
  is shrunk in-place and a new COMPLETED run holds the remainder. Runs are never merged back —
  if a line is removed from a shipment, the production run stays as-is (with whatever reduced
  `rolls_actual`) and a separate remainder run exists. Increasing a line's rolls is not supported
  in edit/ship flows (too complex to reconstruct which remainder run to pull from).

- **Edit vs Ship pages are separate routes**: Edit allows full restructuring (add/remove lines,
  adjust rolls, change ship date). Ship is a lighter confirmation page (roll adjustment only,
  no adding/removing lines) that also commits the status change. This matches the two distinct
  intents.

- **Single unified table on edit**: Current lines and available runs merged into one table rather
  than two separate cards, matching the new-shipment form's feel.
