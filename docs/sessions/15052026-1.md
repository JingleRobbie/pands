## Decisions Made

- Dark mode is **class-based** (`darkMode: 'class'` in Tailwind), toggled per-user stored in DB, applied via `$derived` in layout (not `$state(untrack())` — needed for reactivity after `use:enhance` form saves).
- Dark overrides for complex table selectors use `!important` (PostCSS dev server and Tailwind layers conflict in practice).
- Global `.dark { color: ... }` set as default — catches all unclassed elements instead of hunting individual tags.
- Global `.dark .bg-gray-50/100/white` and `.dark .border-gray-100/200` overrides avoid touching 30+ templates.
- PO edits limited to `expected_date` only — line items must stay in sync with QuickBooks imports.
- Migration tracking via `schema_migrations` table — future pull workflow: `npm run migrate <file>` applies + records.
- **Never use PowerShell `Get-Content`/`Set-Content` for bulk file edits** — use Python for UTF-8 safe replacements.
