## Decisions Made

- **WO ≠ SO:** The CSV previously called "so-update" is actually a work order. WOs are the source document; SOs will be derived later.
- **`so_number` not `wo_number`:** The `work_orders` table uses `so_number` as its unique identifier (matching the `sonumber` CSV column), one record per unique so_number.
- **WO lines store cut-level detail:** `work_order_lines` keeps qty, length_ft, sqft, rollfor, instructions — fields the current `sales_order_lines` model can't hold.
- **No UNIQUE constraint on (wo_id, sku_id):** Same SKU can appear multiple times per WO (different rollfor/length), unlike SO lines.
- **MySQL password changed:** From `Tak3me2$hurch` to `Tak3me2Church` — `.env` already reflects the new value correctly.
